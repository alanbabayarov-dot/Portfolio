## 🎨 2. Как работают кейсы в одностраничнике

Все 6 главных кейсов **внутри одного HTML-файла**, разделены секцией `<article class="case">`. Клиент скроллит — и видит всё. Никаких отдельных страниц, никаких кликов.

Каждый кейс содержит:

1. **Hero-блок** (`.case__hero`) — большое фото или видео 16:9
2. **Head** — заголовок, теги, номер
3. **Body** — описание (2 колонки: текст + таблица фактов)
4. **Gallery** — 2 или 3 изображения 4:5
5. **Опционально:** цитата (`.case__quote`) — есть у EULER

Чтобы поменять контент кейса, ищи в коде комментарий `<!-- CASE 001 — OLYMPIC GAMES -->` и редактируй блок.

## 🖼️ 3. Как добавить изображение в кейс

Найди в коде блок с плейсхолдером:

```html
<div class="case__hero" data-num="N° 002 / EULER" data-empty></div>
```

Замени на:

```html
<div class="case__hero" data-num="N° 002 / EULER">
  <img src="works/002-euler/hero.jpg"
       alt="EULER — Brand DNA"
       loading="lazy" decoding="async">
</div>
```

**Важно:** убери атрибут `data-empty` — это уберёт текст-плейсхолдер `[ HERO 16:9 ]`.

То же самое для галерейных фреймов:

```html
<!-- было -->
<div class="frame" data-cap="LOOK 01" data-empty></div>

<!-- стало -->
<div class="frame" data-cap="LOOK 01">
  <img src="works/001-olympics/look-01.jpg"
       alt="Look 01"
       loading="lazy" decoding="async">
</div>
```

### Рекомендуемые размеры

| Тип | Соотношение | Размер | Вес |
|---|---|---|---|
| Hero кейса (`case__hero`) | 16:9 | 2400×1350px | <500КБ |
| Галерея кейса (`frame`) | 4:5 | 1600×2000px | <300КБ |
| Архивная плитка (`tile__media`) | 4:5 | 800×1000px | <150КБ |

**Оптимизация:** прогоняй все JPG/PNG через **squoosh.app** (бесплатно, в браузере). Уменьшит вес в 3–5 раз без потери качества. Это критично для скорости сайта.

## 🎬 4. Как добавить видео в кейс

### Лёгкое видео (<10 МБ) — прямо в hero

```html
<div class="case__hero" data-num="N° 004 / AI-PIPELINE">
  <video src="works/004-ai/hero.mp4"
         poster="works/004-ai/poster.jpg"
         autoplay muted loop playsinline
         preload="metadata">
  </video>
</div>
```

**Что значат параметры:**
- `poster` — превью-картинка (мгновенный первый кадр, без неё будет чёрный экран)
- `autoplay muted loop` — автопроигрывание в петле без звука (это требование браузеров)
- `playsinline` — на iPhone не открывается на весь экран
- `preload="metadata"` — грузится только первый кадр, остальное по требованию

### Тяжёлое видео (>10 МБ) — через Vimeo

1. Заливаешь на vimeo.com (бесплатно до 500 МБ/неделя)
2. Privacy → Hide from Vimeo
3. Копируешь embed:

```html
<div class="case__hero" data-num="N° 002 / EULER">
  <iframe
    src="https://player.vimeo.com/video/XXXXXXXX?background=1&autoplay=1&loop=1&muted=1"
    frameborder="0" allow="autoplay; fullscreen"
    style="width:100%; height:100%; position:absolute; inset:0;">
  </iframe>
</div>
```

Параметр `background=1` убирает все контролы — выглядит как нативное видео.

### Sora / Kling / Runway

Это `.mp4` — заливай как обычное видео. **Совет:** перед заливкой прогоняй через **HandBrake** (бесплатно):
- Кодек: H.264
- Битрейт: 4–8 Mbps
- Аудио: убрать

Это уменьшает файл в 5–10 раз.

## ⚡ 5. Что уже встроено для скорости

Не надо ничего настраивать дополнительно — это уже работает:

| Фишка | Что делает |
|---|---|
| `loading="lazy"` на img | Грузит картинку только когда она в зоне видимости |
| `preload="metadata"` на video | Грузит только первый кадр видео |
| `IntersectionObserver` для видео | Когда видео уходит за экран — паузится, экономит трафик и батарею |
| Шрифты с `display=swap` | Текст виден сразу, шрифт подгружается отдельно |
| Inline CSS | Без отдельных запросов, всё в одном файле |
| Без JS-библиотек | Никаких jQuery/React — только нативный JS |

## ✏️ 6. Как редактировать текст

### Что менять в `portfolio.html`

| Где | Что | Поиск в коде |
|---|---|---|
| `<title>` | Заголовок вкладки браузера | `<title>` |
| Hero title | Гигантский заголовок (Alan Babayarov) | `class="hero__title"` |
| Hero profile | Bio-описание | `// PROFILE` |
| Кейсы | Тексты, теги, факты | `<!-- CASE 00X -->` |
| Манифест | 4 принципа | `class="principle"` |
| Клиенты | Имена + статусы | `class="client-row"` |
| Услуги | Описания + цены | `class="service"` |
| Email | `hello@yourdomain.com` | поиск `hello@` |
| Соцсети | Ссылки | `class="contact__socials"` |

### Редактор кода

Скачай **VS Code** (бесплатно): https://code.visualstudio.com

Открой папку с портфолио (File → Open Folder). Слева видишь все файлы. Подсветка синтаксиса включается автоматически.

**Чтобы видеть изменения:**
1. Сохрани файл (Cmd+S / Ctrl+S)
2. Открой `portfolio.html` в Chrome/Safari двойным кликом
3. Обнови страницу (Cmd+R / Ctrl+R)

**Pro-совет:** установи расширение **Live Server** в VS Code — автоматически обновляет браузер при каждом сохранении.

## 🌐 7. Как захостить (бесплатно)

### Netlify Drop — самый простой способ

1. Идёшь на **https://app.netlify.com/drop**
2. Перетаскиваешь папку `my-portfolio/` прямо в окно браузера
3. Через 10 секунд получаешь URL: `alan-babayarov.netlify.app`
4. Делишься ссылкой

**Обновить сайт:** просто перетаскиваешь папку заново — старая версия заменится.

### Свой домен (необязательно)

1. Покупаешь на **Porkbun.com** или **Namecheap.com** ($10–12/год)
   - Хорошие варианты: `alanbabayarov.com`, `visualkan.com`, `visualkan.studio`
2. Netlify → Domain settings → Add custom domain
3. Netlify даёт 2 NS-записи — копируешь в настройки домена
4. Ждёшь 1–24 часа
5. HTTPS подключается автоматически

### Что Netlify даёт бесплатно

- HTTPS-сертификат (без него Chrome покажет «не защищено»)
- Контактные формы (если решишь добавить)
- Аналитику посещаемости
- 100 ГБ трафика/месяц

Этого хватит на много тысяч просмотров.

## 🆘 10. Если что-то сломалось

**Сайт не открывается на корне домена:** Переименуй `portfolio.html` → `index.html`. Netlify по умолчанию ищет `index.html` как стартовую страницу.

**Картинка не показывается:**
- Проверь имя файла с учётом регистра (`Hero.jpg` ≠ `hero.jpg`)
- Проверь путь: `works/001-olympics/hero.jpg`
- Открой Developer Tools в Chrome (Cmd+Opt+I) → вкладка Console — увидишь ошибку 404

**Видео не играет на iPhone:** Проверь, что есть атрибут `playsinline` — без него iOS открывает на весь экран при автоплее.

**Видео автоплей не работает:** Проверь, что есть `muted`. Без `muted` браузеры блокируют автоплей. Это не баг — это политика Chrome/Safari.

**Слетает дизайн на телефоне:** Проверь, что в `<head>` есть `<meta name="viewport" content="width=device-width, initial-scale=1.0">`. Должно быть по умолчанию.

**Шрифты выглядят странно (видишь Times New Roman):** Нет интернета или Google Fonts недоступен. Шрифты Fraunces + JetBrains Mono грузятся с fonts.googleapis.com.
